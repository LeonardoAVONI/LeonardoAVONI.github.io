Personal Project · 2023–2024

First Drone Development
Practice

From CAD to firmware to first flight — building a quadcopter entirely from scratch.

Overview

With the theoretical design in hand, this phase turned equations into hardware. Every component was selected, assembled, wired, and programmed by hand — including frame design, electronics integration, firmware configuration, and field testing.

Build Phases

1. Frame Design & Manufacturing

CAD design of the frame in FreeCAD / SolidWorks, targeting a 250 mm motor-to-motor configuration. Parts printed in PETG on an FDM printer for the first iteration, with carbon-fibre arms considered for subsequent versions. Weight budget: under 250 g dry (without battery).

2. Electronics Assembly

Component stack: Flight Controller (FC) · Electronic Speed Controllers (ESC, 4-in-1) · BLDC motors · LiPo battery · RC receiver · GPS module. Soldering, wire routing, and vibration isolation of the FC using damping mounts.

3. Firmware & Configuration

Flashed and configured ArduPilot / Betaflight. Accelerometer and gyroscope calibration, motor spin direction verification, radio control (RC) channel mapping, and failsafe setup. PID gains tuned progressively via Blackbox log analysis.

4. Simulation Before Flight

Software-In-The-Loop (SITL) testing in Gazebo to validate the control gains and verify autonomous mode behaviour before risking hardware.

5. Field Testing & Iteration

First hover tests, stability assessment, and iterative PID tuning. Documented crashes and lessons learned — a rite of passage for any drone builder.

Tools & Technologies

FreeCAD / SolidWorks · FDM 3D Printing · ArduPilot / Betaflight · Gazebo SITL · Python (log analysis) · Soldering iron
